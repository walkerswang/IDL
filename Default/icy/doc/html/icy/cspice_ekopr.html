Abstract


   CSPICE_EKOPR opens an existing E-kernel file for reading,
   returning an integer value for the file handle. Use
   the file handle to identify the file to other EK routines.
   Opening an EK file with this routine makes the EK accessible to 
   the CSPICE EK readers:

   For important details concerning this module's function, please refer to
   the CSPICE routine ekopr_c.

I/O

   
      cspice_ekrcec
      cspice_ekrced
      cspice_ekrcei
   
   all of which expect an EK file handle as an input argument.
   The readers enable reading of individual EK column entries.

Examples


      ;;
      ;; Open the file named by EK1, returning the file handle
      ;; associated with the open file to the variable named
      ;; handle.
      ;;
     
      EK1 = "test1.ek"
      
      cspice_ekopr, EK1, handle
   
      ... do stuff, but nothing that alters the value of handle ...
      
      cspice_ekuef, handle
